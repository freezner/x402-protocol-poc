## 이번에 의도적으로 만들지 않은 기능

사용자 요청에 따라, 아래 항목은 별도 확인 전까지 코드를 생성하지 않았습니다.
현재 `/bank-demo`에서는 정적 데모 데이터 또는 기존 x402 트리거 연결만 제공합니다.

### 1. 데모 전용 계좌 상태 API

기획서의 S6, S8, S9, S12를 실제 데이터로 연결하려면 아래 엔드포인트가 필요합니다.

| 엔드포인트 | 용도 |
|-----------|------|
| `GET /api/demo/account` | AI 계좌 잔액, 월 사용액, 카테고리별 사용량, 위임 AI, 최근 거래 조회 |
| `POST /api/demo/payment` | 출장 시나리오 4건 결제 처리 및 거래내역 반영 |
| `POST /api/demo/micropayment` | 기사 5건 마이크로결제 처리 및 콘텐츠 카테고리 반영 |
| `GET /api/demo/report` | 월간 리포트 데이터 조회 |
| `POST /api/demo/reset` | 발표 전 데모 상태 초기화 |

### 2. 서버 인메모리 상태 저장

현재 프로젝트에는 은행 데모 시나리오 전용 상태 저장소가 없습니다.

필요한 상태는 아래와 같습니다.

| 상태 | 설명 |
|------|------|
| 계좌 잔액 | 초기값 ₩1,000,000, 결제마다 차감 |
| 월 사용액 | 총 사용액과 사용률 계산 |
| 카테고리별 사용량 | 교통/숙박/식음료/콘텐츠 등 누적 |
| 최근 거래 목록 | 가맹점, 금액, 시간, 승인 AI, 상태 |
| 위임 AI 정보 | 현재 등록된 AI, 권한 타입, 만료일 |
| 이상거래 트리거 상태 | 정상 결제 후 차단 시나리오 발생 여부 |

### 3. 이상거래 판정 로직

S10의 핵심인 "왜 차단되었는가"를 실제 판정으로 보여주는 기능이 없습니다.

필요한 요소:

| 항목 | 설명 |
|------|------|
| 위험 점수 계산 | 시간, 위치, 가맹점, 허용 카테고리 기반 |
| 차단 사유 생성 | "해외 결제 미허용", "미등록 가맹점", "패턴 이탈" 등 |
| 사용자 액션 처리 | `거부하기`, `1회 승인` 결과 저장 |
| 차단 이력 반영 | S12 리포트 및 손실 방지 수치 연결 |

### 4. 은행형 결제와 기존 x402 결제의 의미 매핑

현재 연결된 실제 결제는 기존 `/api/demo/trigger`가 호출하는 일반 x402 PoC 엔드포인트입니다.
즉, 아래 항목은 아직 없습니다.

| 미구현 항목 | 현재 대체 상태 |
|------------|----------------|
| 출장 결제 4건 각각의 실제 서버 반영 | S8 화면 애니메이션만 제공 |
| 마이크로결제 5건 각각의 실제 서버 반영 | S11 화면 데이터만 제공 |
| 결제 후 S6/S9 자동 동기화 | 화면 내 데모 데이터로 표시 |
| 카드망 미경유 계좌→가맹점 단위 거래 로그 | 기존 x402 응답 로그만 제공 |

### 5. 발표 운영 기능

기획서 부록 기준 발표용 컨트롤은 아직 일부만 구현되었습니다.

| 항목 | 상태 |
|------|------|
| 화면 이전/다음 버튼 | ✅ 구현됨 |
| 특정 핵심 컷 점프 버튼 | ✅ 구현됨 |
| Space 키 진행 | ❌ 미구현 |
| 자동 재생 시퀀스 | ❌ 미구현 |
